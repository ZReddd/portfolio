Thesis

Sensing the world using EM signals: using radar to gain better situational awareness

This thesis focuses on optimizing Range-Doppler Map (RDMap) generation for passive radar systems by transitioning from Python to C++ and leveraging GPU acceleration with CUDA and cuFFT. Using an FFT-based correlation method, we highlight the importance of power-of-2 signal lengths for consistent accuracy across both CPU and GPU platforms.

The results show significant efficiency gains with GPU acceleration, particularly through the advanced capabilities of the cuFFT library. Additionally, our comparative analysis identifies the O3 optimization level as highly effective in most cases.

These insights have broad implications for real-time signal processing, showcasing potential improvements in efficiency and reliability for passive radar systems.

Passive Radar System

Passive radar systems detect and process signals from external sources, such as cell phone signals or other electromagnetic waves, that are reflected by objects in the environment. By analyzing these reflected signals, passive radar can determine the location, speed, and direction of objects without emitting any signals of its own.

Unlike traditional radar systems, which transmit and receive their own signals, passive radar uses existing electromagnetic waves. It typically employs two channels: a reference channel, which captures direct signals from the source, and a surveillance channel, which detects reflections from objects.

This technology has various applications in surveillance, air traffic control, and other fields, providing an efficient way to monitor environments without actively broadcasting signals.

Understanding CUDA Workflow

This section illustrates the CUDA workflow for optimizing computations on the GPU:

1. Configure memory and initialize data at the host (CPU).
2. Configure memory on the device (GPU).
3. Transfer data from the host to the device (CPU → GPU).
4. Execute the kernel on the GPU.
5. Send data from the device back to the host (GPU → CPU).

Kernel: CUDA enables the creation of custom kernel functions that are executed once per thread, repeated across multiple threads to parallelize tasks efficiently.

Thread Hierarchy: In CUDA, computation is organized into threads. Threads form blocks, and blocks form grids, allowing for scalable parallel processing across the GPU.
